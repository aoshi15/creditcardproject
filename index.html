Credit card fraud detection using SAS

Introduction

Dataset used for this project is the "Credit Card Fraud Detection" dataset available on Kaggle. Below are the details for this dataset:

Credit Card Fraud Detection Dataset:

Description:

  - Contains credit card transactions made by European cardholders in September 2013.

  - Includes a highly imbalanced class distribution, with a small percentage of fraudulent transactions.

  - Features are anonymized due to privacy concerns.

Source:

  - [Credit Card Fraud Detection Dataset on Kaggle](https://www.kaggle.com/mlg-ulb/creditcardfraud)

Attributes:

  - Features V1-V28: Anonymized numerical features resulting from a PCA transformation.

  - Time: Time elapsed between this transaction and the first transaction in the dataset.

  - Amount: Transaction amount.

  - Class: Binary response variable indicating whether the transaction is fraudulent (1) or not (0).

Analyses done

  - Exploratory Data Analysis (EDA) to understand the distribution of features.

  - Handling imbalanced data using techniques like oversampling or undersampling.

  - Building predictive models for fraud detection, such as logistic regression, decision trees, or ensemble methods.

  - Evaluating model performance using appropriate metrics like precision, recall, and ROC-AUC.

Challenges:

  - Imbalanced class distribution may require careful handling during model training.

  - Dealing with anonymized features might pose challenges in feature interpretation.

Handling imbalanced data

We first check the class distribution using `proc freq`. Then, in the `proc iml` block, we use the `smote` function to oversample the minority class (fraudulent transactions). The `Ratio=1` parameter specifies that we want to balance the classes, and `Neighbors=5` defines the number of nearest neighbors to be considered for generating synthetic samples.

Finally, we check the class distribution after oversampling to ensure that the classes are now balanced. Adjust the parameters as needed based on the specific requirements of your project.

/* Check the class distribution */
proc freq data=credit_card;
 tables Class / nocum;
run;

/* Implement SMOTE for oversampling */
proc iml;
 use credit_card;
 read all var {Amount V1-V28 Time Class} into X;
 close credit_card;

 /* Identify the minority class (fraudulent transactions) */
 minorityClass = X[, colname(X, "Class")];

 /* Implement SMOTE */
 call smote(X, minorityClass, Ratio=1, Neighbors=5);

 /* Output the oversampled data */
 create oversampled_data from X;
 append from X;
 close oversampled_data;
quit;

/* Check the class distribution after oversampling */
proc freq data=oversampled_data;
 tables Class / nocum;
run;

Building predictive models for fraud detection

Building models using logistic regression, decision trees, and ensemble methods (random forest) in SAS and comparing their results. We'll use the oversampled dataset from the previous SMOTE example:

  • We split the oversampled dataset into training and testing sets using `proc surveyselect`.
  • We build three different models using `proc logistic`, `proc hpdecisiontree` (SAS Viya's decision tree), and `proc hpforest` (SAS Viya's random forest).
  • The `output` statements save the predicted probabilities for each model.
  • We merge the predictions on the testing set using `merge` and evaluate the model performance using Receiver Operating Characteristic (ROC) analysis with `proc hpbin`.

Adjust the models and parameters based on your specific requirements. The ROC analysis will provide insights into the models' performance in terms of sensitivity and specificity. Further fine-tune the models, consider feature importance, and explore other evaluation metrics based on your project goals.

/* Split the data into training and testing sets */
proc surveyselect data=oversampled_data out=training_data outall
 seed=12345 /* Set a seed for reproducibility */
 samprate=0.7 /* Specify the sampling rate for training set */;
run;

data testing_data;
 set oversampled_data;
 if _selected_ = 0;
run;

/* Logistic Regression Model */
proc logistic data=training_data;
  model Class(event='1') = Amount V1-V28 Time;
  output out=logistic_output predicted=p_logistic;
run;

/* Decision Tree Model */
proc hpdecisiontree data=training_data;
  target Class / level=NOMINAL;
  input Amount V1-V28 Time;
 id transaction_id;
 partition fraction(validate=0.3);
 output out=decision_tree_output predicted=p_decision_tree;
run;

/* Random Forest (Ensemble) Model */
proc hpforest data=training_data;
  target Class / level=NOMINAL;
  input Amount V1-V28 Time;
 id transaction_id;
 partition fraction(validate=0.3);
 output out=random_forest_output predicted=p_random_forest;
run;

/* Assess the model performance on the testing set */
data testing_data_predicted;
 merge testing_data logistic_output decision_tree_output random_forest_output;
  by transaction_id;
run;

/* Assess model performance */
proc hpbin data=testing_data_predicted;
  target Class;
 input p_logistic p_decision_tree p_random_forest;
 roc p_logistic p_decision_tree p_random_forest / sensitivityspecificity=roc_ss;
run;

Evaluating model performance using precision, recall, and ROC-AUC 

  • We use the `roc` statement within `proc logistic` and `proc hpbin` to calculate ROC-AUC, precision, recall, and other related metrics for each model.
  • The `nocut` option specifies that we want to consider all possible cutoff points for calculating ROC curves and related metrics.
  • The `ctable` option provides a table with sensitivity, specificity, precision, and recall values at various cutoff points.
  • The `ci` option adds confidence intervals to the ROC curves.

/* Calculate precision, recall, and ROC-AUC for logistic regression */
proc logistic data=testing_data_predicted;
 model Class(event='1') = p_logistic;
  roc 'Logistic Regression' p_logistic / 
   nocut
    ctable 
   ci
    aggregate;
  ods select ROCInfo;
run;

/* Calculate precision, recall, and ROC-AUC for decision tree */
proc hpbin data=testing_data_predicted;
  target Class;
 input p_decision_tree;
 roc 'Decision Tree' p_decision_tree / 
   nocut
    ctable 
   ci
    aggregate;
  ods select ROCInfo;
run;

/* Calculate precision, recall, and ROC-AUC for random forest */
proc hpbin data=testing_data_predicted;
  target Class;
 input p_random_forest;
 roc 'Random Forest' p_random_forest / 
   nocut
    ctable 
   ci
    aggregate;
  ods select ROCInfo;
run;
